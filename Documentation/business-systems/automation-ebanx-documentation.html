Automation Ebanx

Daily Snowflake-to-Slack notification pipeline for EBANX invoice finalization events

Recipe ID: 66379536 Status: Active Created: 2025-09-29 Last Updated: 2025-09-29 Last Run: 2026-04-14 Author: diana.sy Schedule: Daily 10:00 AM PT Success Rate: 98.0% (201 / 205) Lifetime Tasks: 1,025
SECTION 1: EXECUTIVE SUMMARY

1. Executive Summary

The Automation Ebanx recipe (ID 66379536) is a Workato integration that provides automated daily Slack notifications for newly finalized invoices associated with EBANX payment-method customers. EBANX is a Latin American payment gateway used by Intercom to process payments in local currencies for customers in Brazil and other LATAM markets.

Every day at 10:00 AM Pacific, the recipe queries a Snowflake view (SHARED_EXTERNAL_WORKATO.AUTOMATION_EBANX) for invoices finalized since the last successful run. For each matching invoice, it posts a rich Slack notification to a dedicated channel with invoice details, a direct link to the Stripe invoice dashboard, and customer metadata. After processing, it updates a Workato Lookup Table to record the current date as the new high-water mark.

Since its creation in September 2025, the recipe has processed over 1,000 tasks with a 98% success rate, running autonomously every day. It is a standalone recipe with no child recipes.

SECTION 2: BUSINESS OVERVIEW

2. Business Overview

Business Problem

EBANX customers pay through alternative payment methods (Boleto Bancario, PIX, local credit cards) that have unique settlement timelines and finalization patterns compared to standard Stripe card payments. The Finance team needs real-time visibility into when EBANX invoices are finalized so they can:

  • Track LATAM payment collection in near-real-time
  • Identify and escalate stalled or high-value EBANX invoices promptly
  • Reconcile EBANX payment flows against Stripe records without manually checking dashboards
  • Maintain audit trails of finalized invoice notifications for compliance

SECTION 4: ARCHITECTURE OVERVIEW

4. Architecture Overview

The Automation Ebanx recipe follows a straightforward Extract-Notify-Bookmark pattern:

  1. Bookmark Read -- On trigger, it reads the last_successful_run date from a Workato Lookup Table ("Automations" table, "Automation_Ebanx" row). This serves as the incremental high-water mark.
  2. Extract -- It executes a parameterized SQL query against Snowflake to retrieve all EBANX invoices finalized on or after the high-water mark date.
  3. Notify (Loop) -- For each invoice row returned, it posts a formatted Slack message to a dedicated channel with invoice ID, amount, customer info, dates, and a direct Stripe dashboard link.
  4. Bookmark Update -- After all notifications are sent, it searches the Lookup Table for the "Automation_Ebanx" entry and updates last_successful_run to today's date.

There are no child recipes, retry logic, or error-handling branches. The recipe is a single linear flow with one for-each loop. The architecture is intentionally simple, reflecting its role as a monitoring/alerting automation rather than a transactional integration.

SECTION 9: KEY BUSINESS LOGIC

9. Key Business Logic

Incremental Processing (High-Water Mark Pattern)

The recipe uses a high-water mark (HWM) pattern to ensure incremental processing. The last_successful_run date stored in the Workato Lookup Table ("Automations" / "Automation_Ebanx") is used as the lower bound in the SQL WHERE clause. After successful execution, the HWM is updated to today's date, so the next run only picks up invoices finalized since today.

Important: The WHERE clause uses >= (greater-than-or-equal), meaning invoices finalized on the HWM date itself will be re-fetched on the next run if they were finalized after the recipe ran. This creates a small overlap window that may produce duplicate Slack notifications for invoices finalized on the same day as the last run but after 10:00 AM PT.

SQL Query Logic

SELECT TOTAL_AMOUNT, INVOICE_ID, CUSTOMER_ID, CUSTOMER_NAME,
       INVOICE_DATE, INVOICE_FINALIZED_AT
FROM SHARED_EXTERNAL_WORKATO.AUTOMATION_EBANX
WHERE TO_DATE(INVOICE_FINALIZED_AT) >= '{Last_successful_run}'
  • TO_DATE(INVOICE_FINALIZED_AT) casts the timestamp to a date for comparison against the HWM date
  • The AUTOMATION_EBANX view is maintained in the SHARED_EXTERNAL_WORKATO schema, which is a Snowflake schema shared with Workato for integration use
  • No deduplication or idempotency key is applied at the SQL level -- all matching rows are returned and notified

Slack Message Format

Each Slack notification includes:

  • Bold header text: "New Invoice was created for Ebanx customer - {invoice_id}"
  • Attachment title: Links directly to the Stripe invoice page at dashboard.stripe.com/invoices/{invoice_id}?switch=true
  • Green color bar: Uses Slack's "good" color indicator
  • Structured attachment fields: Invoice ID, Total Amount (prefixed with $), Customer ID, Customer Name, Invoice Date, Invoice Finalized Date

SECTION 11: EDGE CASES

11. Edge Cases

#ScenarioCurrent BehaviorRisk LevelRecommendation
1 Duplicate Notifications -- The >= operator on the HWM date can re-fetch invoices from the same day if they were finalized before 10 AM PT and the HWM date matches. Same invoices are re-notified in Slack. No deduplication. Low Accept as-is (informational only) or switch to timestamp-level HWM for precision.
2 Empty Result Set -- No EBANX invoices were finalized since the last run. The for-each loop is skipped; no Slack messages are sent. The HWM is still updated to today. None No action needed. This is normal weekend/low-volume behavior.
3 Lookup Table Entry Missing -- The "Automation_Ebanx" row in the Automations table is deleted or renamed. Step 1 returns nil for Last_successful_run. The SQL query may fail or return all rows depending on how Snowflake handles the nil date. Medium Add a conditional check: if Last_successful_run is nil, default to (today - 1 day) or halt the recipe.
4 Snowflake View Missing/Changed -- The AUTOMATION_EBANX view is dropped or its schema changes. The SQL execution step will fail, causing the job to error. HWM is NOT updated (correct behavior). Medium Monitor for SQL errors; ensure view changes are coordinated with recipe updates.
5 Slack Channel Archived/Deleted -- Channel C09J6GGPEQH is archived or removed. Slack API will return an error on the first message post, potentially failing the job inside the loop. Medium Configure Workato error notifications; ensure channel archival is coordinated with recipe owners.
6 Large Volume Spike -- An unusually high number of EBANX invoices are finalized (e.g., backlog processing). Sequential loop processes all rows. Slack rate limits may cause failures if hundreds of messages are sent rapidly. Low-Medium Consider adding batch-mode Slack messaging or a summary message for high-volume days.
7 Recipe Stopped and Restarted After Gap -- Recipe is stopped for multiple days then restarted. The HWM was frozen at the date the recipe last ran. On restart, all invoices finalized during the gap will be picked up and notified. Low This is correct catch-up behavior. Expect a burst of Slack messages on restart.
8 HWM Update Fails After Notifications -- Lookup table update (step 7-8) fails but Slack messages were already sent. Next run will re-query the same date range, producing duplicate Slack notifications. Low Accept as low risk since this is a notification-only recipe, not a transactional integration.
