====== 4. ARCHITECTURE OVERVIEW ======

4. Architecture Overview

This integration follows a classic Extract-Transform-Load (ETL) pattern orchestrated through Workato:

EExtract -- NetSuite Web Query

An HTTP GET request hits NetSuite's /app/reporting/webquery.nl endpoint, which returns a saved search report as an HTML table. The request authenticates with entity/email/role parameters and requests CSV format with up to 2,400 rows.

TTransform -- Ruby HTML-to-CSV Conversion

Custom Ruby code uses Nokogiri to parse the HTML response, iterates over all <tr>/<td> elements, strips whitespace and equals signs, and generates a clean CSV string.

LLog -- Debug Output

The transformed data is logged to the Workato job report for debugging and audit purposes. External log forwarding is disabled.

LLoad -- Pigment CSV Push

An HTTP POST sends the CSV payload to Pigment's /api/import/push/csv endpoint with Content-Type text/csv and a configurationId header that maps the data to the correct Pigment import configuration.

Invocation Model

This child recipe exposes a webhook endpoint (Pigment-Trigger) and is invoked by the parent Scheduled Job Pigment (61727250), which runs on a cron schedule 0 6,12,18 * * * (6 AM, 12 PM, 6 PM Pacific). The parent contains identical logic and can also run independently. The child serves as a callable entry point for ad-hoc or alternative triggers.

require 'nokogiri' require 'csv' html = input[:html_table] doc = Nokogiri::HTML(html) # Extract all rows and cells from HTML table rows = doc.css('tr').map do |tr| tr.css('td').map { |td| td.text.strip.gsub('=', '') } end # Generate CSV from extracted rows csv_string = CSV.generate do |csv| rows.each { |row| csv << row } end { csv: csv_string }
Key Detail: The gsub('=', '') call strips equals-sign prefixes that NetSuite adds to text fields in its CSV/HTML export (e.g., ="12345" becomes "12345"). This is a common NetSuite export quirk.

====== 9. KEY BUSINESS LOGIC ======

9. Key Business Logic

9.1 NetSuite Report Extraction

  • The recipe targets a specific NetSuite saved search (entity 431348) in company 3840170.
  • Authentication is done via URL parameters (email + role), leveraging the REST connector's stored credentials.
  • The cr=2400 parameter caps the result set at 2,400 rows, which should accommodate the financial report size.
  • Despite requesting format=csv, NetSuite's Web Query API returns an HTML table structure, necessitating the HTML-to-CSV transformation step.

9.2 HTML-to-CSV Transformation

  • Nokogiri Parsing: The HTML response is parsed with Nokogiri, a robust Ruby HTML/XML parser.
  • Cell Extraction: All <tr> rows are iterated; within each, <td> cells are extracted.
  • Data Cleaning: Each cell value is stripped of whitespace (.strip) and equals signs (.gsub('=', '')). The equals-sign removal is critical because NetSuite prefixes text fields with = to prevent Excel from auto-formatting numbers.
  • CSV Generation: Ruby's built-in CSV library generates a properly escaped, comma-delimited string.

9.3 Pigment Data Push

  • The CSV payload is sent as an application/octet-stream body (with Content-Type: text/csv header override).
  • The configurationId header maps the incoming CSV to a specific import configuration in Pigment, which defines column mappings, data types, and the target dataset/model.
  • The import is a push-based mechanism -- data overwrites or appends based on the Pigment configuration settings (managed within the Pigment platform, not in Workato).

9.4 Dual-Recipe Architecture

  • The parent "Scheduled Job Pigment" (61727250) contains identical business logic and runs on a fixed cron schedule.
  • This child recipe exists as a webhook-callable duplicate, allowing ad-hoc execution without modifying the scheduler.
  • Both recipes share the same REST connector accounts (17229555 for NetSuite, 17229556 for Pigment).

====== 11. EDGE CASES ======

11. Edge Cases

#ScenarioImpactCurrent HandlingRisk
1 NetSuite session/auth expiry Step 1 returns non-200 response or login page HTML HTTP errors are NOT ignored (ignore_http_errors = false), so the job will fail Medium
2 NetSuite report exceeds 2,400 rows Data truncation -- Pigment receives incomplete dataset No overflow detection or pagination; silently truncated at row limit Medium
3 HTML table structure changes Nokogiri parser may extract incorrect data or empty rows No schema validation on parsed output High
4 NetSuite returns empty report Empty CSV pushed to Pigment -- may clear existing data No empty-result guard; proceeds with empty payload Medium
5 Pigment API downtime or rate limit Step 4 HTTP POST fails No retry logic; job will fail and require manual rerun Low
6 Pigment configurationId invalidated Import rejected -- data not loaded Hard-coded UUID in recipe; requires recipe update if Pigment config changes Low
7 Special characters in cell data Malformed CSV if unescaped commas, quotes, or newlines exist Ruby CSV.generate handles escaping, but the .gsub('=', '') could strip legitimate = in formulas Low
8 Concurrent execution (parent + child) Duplicate data push to Pigment if both fire simultaneously No mutex/lock; Pigment idempotency depends on its import configuration Low

12.7 Recommendations

Improvement Opportunities:
  1. Add row count validation: After parsing HTML, verify that the row count is > 0 and < 2,400 (the cap). Alert if the result is empty or exactly at the limit (possible truncation).
  2. Enable error notifications: Configure Workato error handler blocks or external alerting for faster incident response.
  3. Consolidate to single recipe: Consider whether the dual-recipe architecture (parent + child with identical logic) is still needed. The parent handles 97% of executions; the child may be redundant.
  4. Parameterize the saved search ID: Move entity=431348 and cr=2400 to recipe parameters for easier configuration changes.
  5. Add retry logic: Implement error handling with retry for transient NetSuite or Pigment API failures.
