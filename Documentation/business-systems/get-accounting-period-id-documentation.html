1. EXECUTIVE SUMMARY

1. Executive Summary

Get Accounting Period ID is a reusable Workato Recipe Function that accepts a date string as input and returns the corresponding NetSuite accounting period internal ID. It acts as a shared lookup service called by parent recipes in the Rocketlane billing integration pipeline, enabling accurate period assignment when creating or updating financial transactions in NetSuite.

The recipe searches all non-quarter, non-year accounting periods in NetSuite, iterates through results to find the period whose start and end dates bracket the input date, and returns the matching internal ID. It is a critical component in the Rocketlane time-tracking-to-NetSuite billing workflow.

2. BUSINESS OVERVIEW

2. Business Overview

Business Context

Intercom's Professional Services team tracks billable time in Rocketlane, a project management platform. Each time entry must be posted to NetSuite as a financial transaction (journal entry or vendor bill) with the correct accounting period assigned. NetSuite requires transactions to reference a valid accounting period internal ID rather than a raw date.

Problem Solved

This recipe eliminates manual accounting period lookup by providing an automated, callable service that maps any given date to the correct NetSuite accounting period ID. Without it, parent recipes would each need to implement their own period-matching logic, leading to duplication and inconsistency.

Business Value

  • Accuracy: Ensures every Rocketlane billing transaction is posted to the correct accounting period in NetSuite.
  • Reusability: Acts as a shared function callable by any recipe needing period resolution.
  • Compliance: Prevents transactions from being posted to incorrect or closed periods, supporting financial close integrity.
  • Efficiency: Saves manual lookup effort across the billing pipeline.

9. KEY BUSINESS LOGIC

9. Key Business Logic

Core Date-to-Period Matching Algorithm

The recipe implements a linear scan through NetSuite accounting periods to find the one that brackets the input date:

FOR EACH period IN NetSuite_Accounting_Periods:
    IF (input_date.to_date() >= period.startDate.to_date())
       AND (input_date.to_date() <= period.endDate.to_date()):
        accounting_period_id = period.internal_id
        LOG "SUCCESS FOUNDRECORD" (INFO)
    END IF
END FOR

RETURN { accounting_period_id: accounting_period_id }

Search Filter Logic

  • isQuarter = false: Excludes quarterly rollup periods (e.g., "Q1 2026") to avoid ambiguous matches.
  • isYear = false: Excludes yearly rollup periods (e.g., "Year 2026") for the same reason.
  • startDate >= 2023-01-17: Only returns periods from Jan 2023 onward, reducing the search space and improving performance.
  • Limit = 60: Returns up to 60 periods -- approximately 5 years of monthly periods from the start date.

Date Format Acceptance

The recipe comment notes: "Accepts date 01-01-2025 or 01/01/2025 or any like that and returns accounting period internal id." Workato's .to_date() method handles multiple date formats automatically.

Important: The algorithm does not break on first match -- it continues iterating through all periods. If multiple periods overlap (which should not happen in a properly configured NetSuite instance), the last matching period's ID would be returned.

11. EDGE CASES

11. Edge Cases

ScenarioBehaviorRisk Level
Date before 2023-01-17 No matching period found. Returns nil/null for accounting_period_id. High
Date beyond available periods If the date is in a future month beyond the 60-period limit, no match is found. Returns nil. Medium
Invalid date format Workato's .to_date() may raise an error, causing the job to fail. This accounts for some of the 30 failed jobs. High
Closed accounting period The recipe returns the period ID regardless of whether the period is open or closed. The caller is responsible for handling closed periods. Medium
Adjustment periods Although isQuarter and isYear are filtered out, adjustment periods (isAdjust=true) are not excluded. These could create false matches if their date ranges overlap standard periods. Medium
NetSuite SOAP timeout If the NetSuite search times out or returns an error, the recipe fails. No try/catch block is present in this recipe. Medium
Multiple matching periods The loop does not break on first match. If overlapping periods exist, the last match wins. Low
Null Date input Passing null/empty Date will cause .to_date() to fail, resulting in a job error. High

Calling Pattern

  • Root Recipe: Rec 11.0 Rocketlane (70830772) triggers monthly on the 7th at 05:00 AM.
  • Direct Caller: Rocketlane-Child (70830773) calls this recipe synchronously for each Rocketlane time entry.
  • Invocation Volume: Called once per Rocketlane row processed. Batch runs can produce multiple calls in quick succession (observed ~10-second intervals).

Monitoring Recommendations

Failure Rate: The 26.3% failure rate (30/114 jobs) is elevated. Investigate common failure causes -- likely date format issues or NetSuite SOAP errors. Consider adding a try/catch block and error logging to Slack.
  • Period Coverage: The 60-record limit with a start date of 2023-01-17 covers through approximately mid-2028. Monitor and extend the start date filter or increase the limit as time progresses.
  • Logs: Success matches are logged to the Workato log service at INFO level. Debug logging (step 4, step 8) is only in the job report, not the log service.
  • No Error Handling: The recipe has no try/catch block. NetSuite SOAP errors will cause unhandled failures.
  • Recipe must remain running: As a callable function, this recipe must be in "running" state for parent recipes to invoke it successfully.

Improvement Opportunities

  1. Add try/catch: Wrap the NetSuite search in error handling to gracefully report failures to Slack.
  2. Filter adjustment periods: Add isAdjust=false to the search filter to avoid potential false matches.
  3. Break on first match: Consider using a "stop loop" condition after the first match to improve performance.
  4. Validate input date: Add an upfront check that the Date parameter is non-null and parseable before proceeding with the NetSuite search.
  5. Extend period window: Consider removing the fixed start date filter or making it dynamic to future-proof the recipe.
