3.0 Master Recipe - Refunds, Uncollectible Invoices, and Contract Cancellations

Workato Recipe 58628333 · NetSuite ARM Integration · Author: venkatesh.manohar · Generated: 2026-04-01

Purpose

This master orchestration recipe coordinates four financial reconciliation sub-flows between Stripe, Snowflake, and NetSuite. It processes refunds, voided invoices, uncollectible invoices, and cancelled sales orders on a daily basis, ensuring that all financial adjustments originating in Stripe are properly reflected as journal entries, credit memos, and order cancellations in NetSuite.

2. Architecture Diagram

The master recipe acts as an orchestrator, calling four child recipes sequentially. Each child recipe queries Snowflake staging tables for unprocessed records and then iterates through them, calling grandchild "RF" recipes that create the actual NetSuite transactions.

Step Flow

  1. Trigger: Receives Start_date parameter from master recipe.
  2. Snowflake Query: Queries NETSUITE_EXPORT_REFUNDS joined with NETSUITE_EXPORT_CUSTOMER and NETSUITE_EXPORT_PAYMENT, excluding records already logged in WORKATO_LOGS.
  3. For Each Row: Iterates through unprocessed refund rows.
  4. Call RF 8.0: Passes Customer, Subsidiary, Date, Stripe_Refund_ID, Stripe_Charge_ID, Stripe_Invoice_ID, Stripe_Credit_Note_ID, Stripe_Automatic_Payout_ID, Refund_Amount, external_id.
View SQL Query (Snowflake) 
WITH stg_workato_log AS (
    SELECT
        external_id,
        job_run_completed_at AS last_successful_job_run_completed_at,
        netsuite_internal_id,
        source_table_name,
        error_type,
        error_message
    FROM intercom_prod.shared_external_workato.WORKATO_LOGS
    WHERE
        external_id IS NOT NULL
        AND source_table_name = 'SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_REFUNDS'
        AND environment_type = 'prod'
        AND is_successful_write_to_netsuite = true
    QUALIFY ROW_NUMBER() OVER (PARTITION BY external_id ORDER BY job_run_completed_at DESC) = 1
)
SELECT
    r.customer,
    stripe_refund_id,
    r.subsidiary,
    credit_note_id,
    c.category,
    r.stripe_invoice_id,
    stripe_charge_id,
    stripe_automatic_payout_id,
    refund_amount,
    r.date,
    r.created_date,
    r.netsuite_internal_id
FROM
    SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_REFUNDS r
LEFT JOIN
    SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_CUSTOMER c ON r.customer = c.external_id
LEFT JOIN
    SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_PAYMENT p ON r.stripe_charge_id = p.STRIPE_PAYMENT_ID
LEFT JOIN
    stg_workato_log wl ON wl.external_id = r.stripe_refund_id
WHERE
    r.date >= '{Start_date}' AND r.date <= DATEADD(day, -1, CURRENT_DATE)
    AND wl.netsuite_internal_id IS NULL
ORDER BY r.created_date

4.2 RF 8.0 - Create Refunds (Grandchild)

Recipe ID: 54665275. This recipe receives individual refund parameters and creates a Customer Refund record in NetSuite. It maps the Stripe refund data to the appropriate NetSuite fields, matching the payment via Stripe Charge ID and applying the refund to the correct customer deposit or payment.

Key Deduplication: The WORKATO_LOGS CTE ensures only unprocessed refunds are selected. Records with a successful netsuite_internal_id in the log are excluded from the query results.

Step Flow

  1. Trigger: Receives Date parameter.
  2. Snowflake Query: Queries NETSUITE_EXPORT_VOIDED_INVOICE joined with NETSUITE_EXPORT_ITEM, excluding already-logged records. Filters on INVOICE_VOIDED_AT >= Date and amount != 0.
  3. Create Unique_Invoice_List: Extracts unique stripe_invoice_id values.
  4. Create List: Creates an iteration list sized to the unique invoice count.
  5. Create Invoices List: Declares a structured list variable with 16 fields (stripe_invoice_id, subsidiary, customer, invoice_voided_at, invoice_total, item, item_description, amount, tax_amount, line_type, actual_qty, discount_percentage, rev_rec_start/end_date, total_tax, item_internal_id).
  6. For Each Unique Invoice: Groups line items by invoice, adds them to the Invoices list, then calls RF 10.1.
  7. Call RF 10.1: Passes grouped invoice header + line items to create credit memos in NetSuite.
  8. Clear Invoices List: Resets the list for the next iteration.
View SQL Query (Snowflake) 
WITH stg_workato_log AS (
    SELECT
        external_id,
        job_run_completed_at AS last_successful_job_run_completed_at,
        netsuite_internal_id,
        source_table_name,
        error_type,
        error_message
    FROM shared_external_workato.WORKATO_LOGS
    WHERE
        external_id IS NOT NULL
        AND source_table_name = 'SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_VOID_INVOICE'
        AND environment_type = 'prod'
    QUALIFY ROW_NUMBER() OVER (PARTITION BY external_id ORDER BY job_run_completed_at DESC) = 1
)
SELECT
    STRIPE_INVOICE_ID, SUBSIDIARY, CUSTOMER, INVOICE_VOIDED_AT,
    INVOICE_TOTAL, ITEM, ITEM_DESCRIPTION, AMOUNT, TAX_AMOUNT,
    LINE_TYPE, ACTUAL_QTY, DISCOUNT_PERCENTAGE, TOTAL_TAX,
    REV_REC_END_DATE, REV_REC_START_DATE
FROM
    SHARED_EXTERNAL_WORKATO.netsuite_export_voided_invoice inv
LEFT JOIN
    SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_ITEM i ON inv.item = i.billing_plan_id
LEFT JOIN
    stg_workato_log AS wl ON inv.stripe_invoice_id = wl.external_id
WHERE INVOICE_VOIDED_AT >= '{Date}'
    AND wl.netsuite_internal_id IS NULL
    AND amount != 0
ORDER BY INVOICE_VOIDED_AT

View SQL Query (Snowflake) 
WITH stg_workato_log AS (

        external_id,

SELECT DISTINCT
    c.salesforce_contract_id,
    c.customer,
    c.cancellation_date
FROM shared_external_workato.netsuite_export_contracts c

    AND wl.netsuite_internal_id IS NULL

10. Error Handling and Alerting

10.1 Master Recipe Error Handling

The master recipe wraps all four child recipe calls in a try/catch block. On any error (typically timeouts from processing too many records), it sends an HTML email alert.

10.2 Child Recipe Error Handling

Each child recipe (Rec 7.0, 10.1, 10.0, 9.0) has its own try/catch block with max_retry_count: 0 and retry_interval: 2s. The catch blocks are empty -- errors propagate up to the master recipe's catch handler.

10.3 Deduplication via WORKATO_LOGS

All flows use the WORKATO_LOGS table as an idempotency mechanism. Each query includes a CTE that checks whether a record's external_id has already been successfully written to NetSuite. Records with a netsuite_internal_id in the log are excluded from processing. This prevents duplicate transaction creation on re-runs.

Pattern: LEFT JOIN stg_workato_log wl ON wl.external_id = ... WHERE wl.netsuite_internal_id IS NULL

12.5 Data Flow Summary

The integration follows a consistent pattern across all flows:

  1. Extract: Query Snowflake staging tables for records not yet in WORKATO_LOGS.
  2. Transform: Map Snowflake fields to NetSuite transaction fields (customer, subsidiary, amounts, dates, line items).
  3. Load: Create transaction in NetSuite via SuiteTalk API (handled by RF recipes).
  4. Log: Write success/failure to WORKATO_LOGS for idempotency.
