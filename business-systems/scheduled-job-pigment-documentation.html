Section 1: Executive Summary

1. Executive Summary

The Scheduled Job Pigment recipe is an automated data pipeline that extracts financial report data from NetSuite's web query API, transforms it from HTML to CSV format using custom Ruby code, and pushes the cleaned data into Pigment (an FP&A/financial planning platform) via its CSV import API. The integration runs three times daily to ensure Pigment's planning models remain synchronized with the latest NetSuite financial actuals.

Section 4: Architecture Overview

4. Architecture Overview

The recipe follows a classic Extract-Transform-Load (ETL) pattern orchestrated by Workato's scheduler. The architecture is linear and single-threaded with no branching, conditionals, or child recipe calls.

Architecture Pattern

  • Pattern: Scheduled ETL (Extract-Transform-Load)
  • Trigger: Cron-based scheduler (3x daily at 6AM, 12PM, 6PM Pacific)
  • Extract: HTTP GET to NetSuite Web Query API returning HTML report data
  • Transform: Custom Ruby code using Nokogiri to parse HTML tables and convert to CSV
  • Load: HTTP POST to Pigment CSV Import API with configuration ID header
  • Observability: Intermediate logging of transformed data to Workato job report

Key Design Decisions

  • Uses NetSuite's web query endpoint rather than SuiteQL/REST for report extraction (leverages existing saved report with complex formatting)
  • HTML-to-CSV transformation done via Ruby/Nokogiri rather than direct CSV export (NetSuite report delivers HTML-formatted data)
  • Non-2xx response handling enabled on both HTTP calls for resilience
  • Single monolithic recipe (no child recipes) for simplicity

Section 6: End-to-End Flow

6. End-to-End Flow

Flow Description

  1. Trigger (Scheduler): Workato's cron scheduler fires at 6:00 AM, 12:00 PM, and 6:00 PM Pacific Time every day.
  2. Step 1 - Extract (NetSuite HTTP GET): The recipe calls NetSuite's web query endpoint (/app/reporting/webquery.nl) with parameters specifying the saved report, company ID, entity, email, role, and format (CSV). Despite requesting CSV format, NetSuite returns HTML-formatted data. The response is captured as raw text (UTF-8).
  3. Step 2 - Transform (Ruby Custom Code): The raw HTML response is passed to a Ruby code block that uses Nokogiri::HTML to parse the HTML, extract all table rows (tr) and cells (td), strip whitespace, remove = characters, and generate a clean CSV string.
  4. Step 3 - Log (Logger): The transformed output is logged to the Workato job report for debugging and audit purposes.
  5. Step 4 - Load (Pigment HTTP POST): The CSV data is sent as a POST request to Pigment's CSV import API (/api/import/push/csv) with the appropriate configurationId header and text/csv content type.

Section 9: Key Business Logic

9. Key Business Logic

HTML-to-CSV Transformation Logic

The core transformation logic uses Ruby with the Nokogiri gem to parse the HTML response from NetSuite:

  1. HTML Parsing: Nokogiri::HTML(html) parses the raw HTML response into a DOM tree
  2. Row Extraction: doc.css('tr') selects all table rows from the HTML
  3. Cell Extraction: tr.css('td') selects all data cells within each row (note: th headers are excluded)
  4. Data Cleaning: Each cell value is processed with:
    • .text - Extracts text content, stripping HTML tags
    • .strip - Removes leading/trailing whitespace
    • .gsub('=', '') - Removes equals signs (NetSuite formula prefix artifacts)
  5. CSV Generation: CSV.generate properly formats the cleaned rows into a valid CSV string with correct escaping

Important Business Rules

  • Report Identity: The NetSuite report is accessed by entity ID 431348 under company 3840170, using role 1025. Any changes to the saved report in NetSuite will directly affect the data pushed to Pigment.
  • Row Limit: The cr=2400 parameter caps the report at 2,400 rows. If the report grows beyond this limit, data will be silently truncated.
  • Equals Sign Removal: The gsub('=', '') cleaning step removes ALL equals signs from data values. This is designed to strip NetSuite's formula prefix characters but could unintentionally modify data containing legitimate equals signs.
  • Header Row Handling: The code uses td selectors only, meaning table header cells (th) are excluded from the output. This assumes Pigment's import configuration expects data without headers, or the NetSuite report does not use th tags.
  • Full Replace Pattern: Each run pushes the entire report to Pigment (full dataset push, not incremental). The Pigment configuration ID determines how the data is mapped and whether it replaces or appends.

Error Handling Strategy

  • Both HTTP steps have ignore_http_errors enabled, meaning non-2xx responses are treated as successful actions and do not halt the recipe
  • No explicit error handling, retry logic, or conditional branching exists in the recipe
  • Failed jobs (24 out of 683 = 3.5% failure rate) are likely caused by NetSuite report timeouts or API unavailability

11. Edge Cases & Risk Factors

Known Risks

RiskSeverityDescriptionMitigation
Row Truncation High If the NetSuite report exceeds 2,400 rows (cr=2400), data is silently truncated with no alert Monitor report row count; increase cr parameter if needed
Equals Sign Removal Medium The gsub('=', '') removes ALL equals signs, potentially corrupting data containing legitimate = characters Review report data for = characters; consider more targeted regex
Silent HTTP Errors Medium Both HTTP steps ignore non-2xx errors, meaning Pigment could receive empty/error data without the recipe failing Add conditional checks on response status codes; validate response body
Header Row Omission Medium Only td elements are extracted (not th), so table headers may be missing from CSV output Verify Pigment import config handles headerless data correctly
Service Account Dependency Medium The recipe authenticates as diana.sy@intercom.io; if this account is deactivated or role 1025 is modified, extraction fails Use a shared service account; document role requirements
NetSuite Report Changes High Any structural changes to the saved report (entity 431348) in NetSuite will change the data pushed to Pigment, potentially breaking the Pigment import mapping Coordinate report changes with Pigment configuration updates
Empty Report Low If NetSuite returns an empty report, the recipe will push an empty CSV to Pigment Add validation for minimum expected row count
