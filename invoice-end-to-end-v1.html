Invoice Pipeline — End-to-End System Documentation

Complete system-level documentation: Stripe → Fivetran → Snowflake (dbt) → Workato → NetSuite

Type: End-to-End Author: venkatesh.manohar Generated: April 9, 2026 Systems: 5 (Stripe, Fivetran, Snowflake, Workato, NetSuite) Workato Recipes: 18 total (10 active, 8 stopped) Schedule: 4x daily (5:30, 10:30, 15:30, 20:30 PT)

1. Executive Summary

Business View

When a customer is billed in Stripe (either self-serve or sales-led), an Invoice is generated that must be recorded in NetSuite (our ERP) so Finance can recognize revenue, manage accounts receivable, and track collections. This pipeline automates the entire flow — from the moment a Stripe invoice is finalized to a fully-formed Invoice (with correct line items, quantities, pricing, rev-rec dates, and tax) appearing in NetSuite.

It also handles uncollectible invoices (write-offs via journal entries), voided invoices (reversals via credit memos), and negative invoice credit memos for proration/downgrade scenarios — all without manual Finance intervention.

Technical View

A 5-system pipeline: Stripe (source of truth for invoices and subscriptions) → Fivetran (data share + CDC replication to Snowflake) → Snowflake (dbt staging/intermediate models + Workato export tables) → Workato (orchestration, business logic, API calls) → NetSuite (destination ERP).

Workato runs on a cron schedule (4x/day), queries Snowflake for unsynced finalized invoices, groups line items by invoice, and creates Invoices in NetSuite via SuiteTalk API. A log table (WORKATO_LOGS) tracks sync state to ensure idempotency.

Reliability: The core child recipe (RF 4.00 Create Invoice - Sales Led) has processed 116,905 jobs with 0 failures. RF 4.01 (Self Serve) has processed 391,192 jobs with 0 failures. The pipeline processes both sales-led and self-serve invoices across all subsidiaries.

2. Business Overview

What Problem Does This Solve?

Intercom's billing system uses Stripe as the payment processor and invoice generator. Every finalized Stripe invoice must be mirrored as a NetSuite invoice so Finance can manage revenue recognition, billing schedules, and accounts receivable. Without automation, Finance would need to manually create thousands of invoices per month, including complex discount calculations, multi-currency handling, and tax allocation across subsidiaries.

Business Flow

  1. Customer is billed in Stripe — A subscription renews or a new invoice is generated for a sales-led or self-serve customer
  2. Invoice is finalized — Stripe finalizes the invoice with line items, amounts, discounts, and tax calculations
  3. Data flows to Snowflake — Fivetran replicates Stripe invoice data to Snowflake via data share and CDC; dbt transforms it through staging/intermediate layers into the export table (NETSUITE_EXPORT_INVOICE)
  4. Workato picks up unsynced invoices — 4x daily, Workato queries Snowflake for finalized invoices not yet synced to NetSuite (via WORKATO_LOGS dedup check)
  5. Invoice created in NetSuite — Workato creates the invoice via SuiteTalk API with all line items, discounts, tax, and rev-rec schedules
  6. Special cases handled — Uncollectible invoices create journal entries; voided invoices create credit memos; negative invoices (proration/downgrade) create standalone credit memos

4. Architecture Overview

Source Layer

Stripe Billing holds the canonical invoice data. When a subscription renews or a customer is billed, Stripe generates a finalized invoice with line items, discounts, tax, and payment information.

Source Layer — Technical

Stripe Data Share provides near-real-time data to Snowflake via Snowflake Data Sharing (no ETL needed). Fivetran CDC also replicates event-level Stripe data for invoice payments and webhooks.

Data Layer

Snowflake serves as the central data platform. A dbt pipeline transforms Stripe data through staging and intermediate layers, then produces multiple export tables that Workato reads — one for standard invoices, one for voided invoices, one for negative invoice credit memos.

Data Layer — Technical

Sequential dbt pipeline: STG_STRIPE_DATA_SHARE_INVOICES (view) → STG_STRIPE_INVOICES (table) → INT_STRIPE_INVOICES (intermediate) → NETSUITE_EXPORT_INVOICE (6.7M rows, 71 cols). V2 variant: EXT_NETSUITE_EXPORT_INVOICE_V2 (6.6M rows, 41 cols).
Workato reads the export table, joined with NETSUITE_EXPORT_ITEM (billing plan → NS item mapping) and cross-referenced against WORKATO_LOGS for dedup.

Orchestration Layer

Workato is the automation engine. It runs 4x daily, finds invoices that need syncing, applies business rules (sales-led vs self-serve routing, discount application, tax handling), and creates the NetSuite records.

Orchestration Layer — Technical

Scheduler (0.0, cron 4x/day) → 1.0 Master Invoice → Rec 4.0 Invoice Master (Snowflake SQL + group by invoice) → RF 4.00 (Sales Led) or RF 4.01 (Self Serve). Parallel branch: 3.0 Master → Rec 10.1 Voided → RF 10.1, and Rec 10.0 Uncollectible → RF 10.0. WORKATO_LOGS table used for idempotent dedup.

Target Layer

NetSuite receives the Invoices, Credit Memos, and Journal Entries. Finance uses these for revenue recognition, AR management, and financial reporting. Custom fields link back to Stripe for traceability.

Target Layer — Technical

NetSuite SuiteTalk API. Invoices use CustInvc record type with custom body/line fields. Voided invoices create CustCred (Credit Memo). Uncollectible invoices create JournlEnt (Journal Entry). External ID = Stripe Invoice ID for idempotent upserts.

6. End-to-End Flow

The complete data journey for an Invoice follows this sequential path across five systems:

  1. Stripe Billing — A subscription renews or a customer is billed. Stripe generates a finalized invoice with line items (product, quantity, amount), discounts (percentage or fixed), tax, and payment information (charge ID, payment intent).
  2. Fivetran / Data Share — Stripe Data Share provides near-real-time access to invoice data in Snowflake. Fivetran also syncs Stripe events for payment status tracking. A separate connector syncs NetSuite data back for reconciliation.
  3. Snowflake dbt Pipeline — A sequential dbt pipeline transforms the raw data:
    • STG_STRIPE_DATA_SHARE_INVOICES — staging view from Stripe data share
    • STG_STRIPE_INVOICES — staging table (dedup, rename, type cast)
    • STG_STRIPE_INVOICE_LINES — staging table for line items
    • INT_STRIPE_INVOICES — intermediate table (enriched with subscription context)
    • NETSUITE_EXPORT_INVOICE — export table (6.7M rows, 71 cols) — this is what Workato reads
    • NETSUITE_EXPORT_VOIDED_INVOICE — export table (6.7K rows) for voided invoices
    • NETSUITE_EXPORT_CREDIT_MEMO_NEGATIVE_INVOICE — export table (2.7K rows) for negative invoice credit memos
  4. Workato Orchestration — Runs 4x daily via cron scheduler. 1.0 Master Invoice calls Rec 4.0 which queries the export table, groups line items by invoice, determines sales-led vs self-serve, and calls the appropriate child recipe (RF 4.00 or RF 4.01). In parallel, 3.0 Master handles uncollectible (RF 10.0) and voided (RF 10.1) invoices.
  5. NetSuite ERP — Receives Invoices (CustInvc), Journal Entries (JournlEnt for uncollectible), and Credit Memos (CustCred for voided) via SuiteTalk API. Revenue recognition begins once the invoice is approved.
Feedback loops: Workato writes sync results (ns_internal_id, status, error_type, error_message) back to WORKATO_LOGS in Snowflake for dedup tracking. Fivetran's NetSuite connector syncs transaction data back to FIVETRAN_NETSUITE for monthly reconciliation. The WORKATO_LOGS source_table_name field distinguishes between invoice types (e.g., SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_VOID_INVOICE).

9. Key Business Logic

Sales Channel Routing

Business View

Invoices are routed to different child recipes based on the SALES_CHANNEL field. Sales-led invoices (from Salesforce contracts) go through RF 4.00, while self-serve invoices (direct Stripe subscriptions) go through RF 4.01. This matters because sales-led invoices carry additional fields like Salesforce Contract ID and custom terms.

Technical View

Rec 4.0 queries NETSUITE_EXPORT_INVOICE with a LEFT JOIN to WORKATO_LOGS (filtering wl.netsuite_internal_id IS NULL for unsynced records). Groups results by STRIPE_INVOICE_ID using .pluck('stripe_invoice_id').uniq. Routes to RF 4.00 or RF 4.01 based on the SALES_CHANNEL value.

Discount Handling

Business View

Invoices can carry both line-level discounts (percentage or fixed amount) and header-level discounts. Stripe tracks DISCOUNT_PERCENTAGE, DISCOUNT_AMOUNT, and DISCOUNT_ITEM. NetSuite needs these mapped to discount line items with the correct NS internal IDs.

Technical View

The export table carries DISCOUNT_PERCENTAGE (FLOAT), DISCOUNT_ITEM (TEXT), TOTAL_DISCOUNT (FLOAT), and AMOUNT_WITHOUT_HEADER_DISCOUNT (FLOAT). Workato maps the billing plan discount item to a NetSuite item ID via the NETSUITE_EXPORT_ITEM lookup table (billing_plan_id → netsuite_internal_id). Friends & family discounts are tracked separately via the FRIENDS_FAM_DISCOUNT_APPLIED boolean.

Uncollectible Invoice Write-off

Business View

When an invoice cannot be collected (customer default), it is marked uncollectible in Stripe. Finance requires a Journal Entry in NetSuite to write off the receivable. This debits bad debt expense and credits accounts receivable.

Technical View

The INVOICE_UNCOLLECTIBLE_AT timestamp in the export table triggers Rec 10.0 / RF 10.0. RF 10.0 creates a JournlEnt in NetSuite with debit/credit lines. Parameters include Subsidiary, customer_id, external_id (Stripe Invoice ID), Invoice_total, and Amount_Due. The base_external_id links back to the original invoice for audit trail.

Voided Invoice Reversal

Business View

When an invoice is voided in Stripe (e.g., billing error, duplicate), a Credit Memo must be created in NetSuite to reverse the original invoice. The credit memo carries the same line items with reversed amounts.

Technical View

Rec 10.1 queries NETSUITE_EXPORT_VOIDED_INVOICE filtering on INVOICE_VOIDED_AT and checking WORKATO_LOGS for unsynced records (source_table_name = SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_VOID_INVOICE). It groups lines by STRIPE_INVOICE_ID, then calls RF 10.1 for each unique invoice. RF 10.1 creates a CustCred in NetSuite with line items mirroring the original invoice. Only non-zero amount lines are processed (amount != 0 filter).

Item Mapping

Business View

Stripe billing plans must be mapped to NetSuite inventory items. The NETSUITE_EXPORT_ITEM table (100 rows) serves as the master mapping table, translating Stripe billing plan IDs to NetSuite internal IDs.

Technical View

The SQL in Rec 4.0 includes LEFT JOIN SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_ITEM i ON inv.item = i.billing_plan_id. The netsuite_internal_id from this join is used as the item reference when creating invoice line items in NetSuite. When no mapping exists, the item field from the export table is used as a fallback identifier.

11. Edge Cases

Known Edge Cases and Handling

Edge CaseWhat HappensHow It Is Handled
Invoice with negative amount (proration/downgrade) Stripe generates an invoice with negative line amounts for mid-cycle downgrades Routed to NETSUITE_EXPORT_CREDIT_MEMO_NEGATIVE_INVOICE table; Workato creates a standalone Credit Memo in NetSuite rather than a standard invoice
Manual invoice (not from Stripe subscription) One-off manual invoices may lack a subscription ID or standard billing plan Manual push recipe (5.02.4) handles ad-hoc syncs by Stripe Invoice ID. Note: CBAs from manual invoices may not appear in NetSuite because the routing debug string in NS ID prevents RF 8.0 from parsing
Missing item mapping New billing plan in Stripe with no corresponding entry in NETSUITE_EXPORT_ITEM LEFT JOIN returns NULL for netsuite_internal_id; Workato uses the raw billing plan ID and may fail at NS API level. Requires manual addition to the 100-row item mapping table
Duplicate invoice sync attempt Same invoice processed in consecutive Workato runs WORKATO_LOGS table tracks external_id per source_table_name. Rec 4.0 query includes WHERE wl.netsuite_internal_id IS NULL to skip already-synced invoices
Multi-subsidiary invoice Customer with invoices across different NetSuite subsidiaries SUBSIDIARY field on each line determines which NetSuite subsidiary receives the record. Workato passes this as a parameter to the child recipe
$0 invoices (100% discount) Full discount results in zero-amount invoice Still synced to NetSuite with correct quantities and rev-rec dates; amount is $0
Timezone / date boundary issues Invoices finalized near midnight may appear in different date windows Extend monthly recon windows by 3 days for Workato/Fivetran sync lag to avoid false mismatches (per operational guidance)
Workato timeout on large batches Too many invoices in a single run causes Workato job timeout Try/catch wrapper in 1.0 Master sends email alert to danny.mcgean, nan.lu, chase.clark, diana.sy with error details. Retry on next scheduled run
Invoice voided after sync to NetSuite Original invoice already exists in NetSuite when void event arrives RF 10.1 creates a reversing Credit Memo referencing the original. The WORKATO_LOGS entry for the void uses a separate source_table_name

12. Operational Notes

Monitoring and Alerting

  • Schedule: 4x daily at 5:30, 10:30, 15:30, 20:30 PT via 0.0 Scheduled Master
  • Error alerting: Email sent to danny.mcgean, nan.lu, chase.clark, diana.sy on timeout errors
  • Dedup tracking: WORKATO_LOGS table in SHARED_EXTERNAL_WORKATO — keyed by external_id + source_table_name
  • Job monitoring: Workato dashboard shows job success/failure counts per recipe

Common Troubleshooting Steps

  1. Invoice not appearing in NetSuite: Check WORKATO_LOGS for the Stripe Invoice ID — if netsuite_internal_id is NULL and error_message is populated, investigate the error. Common causes: missing item mapping, invalid customer, subsidiary mismatch.
  2. Duplicate invoice in NetSuite: Check if WORKATO_LOGS has multiple entries for the same external_id. The QUALIFY ROW_NUMBER() OVER (PARTITION BY external_id ORDER BY job_run_completed_at DESC) = 1 pattern ensures only the latest log is checked.
  3. Manual resync needed: Use recipe 5.02.4 (Invoice Master Recipe- Manual) with the specific Stripe Invoice ID. This bypasses the scheduled run and processes a single invoice on demand.
  4. Uncollectible not creating JE: Check if INVOICE_UNCOLLECTIBLE_AT is populated in the export table. Verify Rec 10.0 is running (currently stopped — superseded by 3.0 Master calling it directly).

Key SQL Pattern (Workato Query)

Rec 4.0 Invoice Master SQL (click to expand) 
SELECT
    inv.customer, inv.subsidiary, inv.date,
    stripe_invoice_id, inv.salesforce_contract_id,
    stripe_invoice_status, inv.sales_channel,
    payment_method, invoice_total,
    inv.item, inv.item_description,
    inv.actual_qty, inv.amount,
    inv.rev_rec_start_date, inv.rev_rec_end_date,
    discount_item, discount_percentage,
    discount_amount, inv.created_date,
    line_type, i.netsuite_internal_id,
    total_tax, amount_due, updated_date
FROM
    SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_INVOICE inv
LEFT JOIN
    SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_ITEM i
    ON inv.item = i.billing_plan_id
WHERE
    stripe_invoice_id = :stripe_invoice_id
ORDER BY updated_date
Rec 10.1 Voided Invoice SQL (click to expand) 
WITH stg_workato_log AS (
    SELECT external_id,
           job_run_completed_at AS last_successful_job_run_completed_at,
           netsuite_internal_id, source_table_name,
           error_type, error_message
    FROM shared_external_workato.WORKATO_LOGS
    WHERE external_id IS NOT NULL
      AND source_table_name = 'SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_VOID_INVOICE'
      AND environment_type = 'prod'
    QUALIFY ROW_NUMBER() OVER (
        PARTITION BY external_id ORDER BY job_run_completed_at DESC
    ) = 1
)
SELECT STRIPE_INVOICE_ID, SUBSIDIARY, CUSTOMER,
       INVOICE_VOIDED_AT, INVOICE_TOTAL, ITEM,
       ITEM_DESCRIPTION, AMOUNT, TAX_AMOUNT,
       LINE_TYPE, ACTUAL_QTY, DISCOUNT_PERCENTAGE,
       TOTAL_TAX, REV_REC_END_DATE, REV_REC_START_DATE
FROM SHARED_EXTERNAL_WORKATO.netsuite_export_voided_invoice inv
LEFT JOIN SHARED_EXTERNAL_WORKATO.NETSUITE_EXPORT_ITEM i
    ON inv.item = i.billing_plan_id
LEFT JOIN stg_workato_log AS wl
    ON inv.stripe_invoice_id = wl.external_id
WHERE INVOICE_VOIDED_AT >= :start_date
  AND wl.netsuite_internal_id IS NULL
  AND amount != 0
ORDER BY INVOICE_VOIDED_AT
